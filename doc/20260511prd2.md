## 1. 产品概述

### 1.1 产品定位
M2是一个面向企业内部的API中转与分发平台，核心目标是为外部组织（客户）提供稳定、可度量、快速发放的DeepSeek V4 Flash模型API访问服务。平台聚焦于Key的高效发放与精准计费，弱化复杂权限层级，以轻量化客户库代替绑定关系。

### 1.2 核心目标
1. **极简管控**：仅保留管理员、授权实施/研发人员两类内部角色，任何授权人员均可直接为客户组织生成、管理Key。
2. **精准计费**：以上游API（DeepSeek）返回的实际Token消耗为唯一计费依据，不加价，直接扣除Key的额度。
3. **高效Key管理**：以Key为中心，支持按客户名称筛选、批量停用/启用，且Key可二次查看，完整展示创建者与归属客户。
4. **试用与正式区分**：明确划分永久Key与测试Key，满足长期合作与临时试用场景。

### 1.3 核心术语定义
| 术语 | 定义 |
|------|------|
| **内部员工** | 平台的操作者，分为管理员和授权实施/研发人员，具有登录后台的权限。 |
| **客户(组织)** | 需要调用API的外部企业或团队，仅作为Key的归属标签，记录在客户库中。客户不可登录后台。 |
| **Key** | 下发给客户组织的API调用凭证，每个Key拥有独立的额度、类型（永久/测试）和状态，是计费与控制的唯一单元。 |
| **永久Key** | 无过期时间、无预设额度上限的Key，适用于正式商业使用。 |
| **测试Key** | 创建时需设定额度（元），额度耗尽或主动停用后不可调用，适用于产品试用。 |
| **Token消耗** | 上游API（DeepSeek）在Response中返回的usage对象，含prompt_tokens和completion_tokens。 |
| **创建者** | 生成该Key的内部员工，用于追溯Key的来源。 |

## 2. 模块一：角色与权限

### 2.1 角色定义
平台仅保留以下三类角色，其中客户不具有登录后台权限：

1. **管理员**：拥有平台全部管理权限。
2. **授权实施/研发人员**：可进行Key生成、客户库维护、用量查看等操作。
3. **客户**：仅通过Key调用API，无后台账号。

### 2.2 权限矩阵
| 关键操作 | 管理员 | 授权实施/研发人员 | 客户 |
|----------|--------|-------------------|------|
| 管理内部员工账号 | **Y** | N | N |
| 维护客户库（增/查） | Y | **Y** | N |
| 生成Key（含永久/测试） | Y | **Y** | N |
| 查看、搜索所有Key | Y | Y（可查看所有Key） | N |
| 批量停用/启用Key | Y | Y（仅可操作自己生成的Key） | N |
| 查看Key详情、复制Key | Y | Y | N |
| 查看全局用量报表 | Y | N | N |
| 查看个人生成Key的用量 | Y | Y | N |
| 使用Key调用API | N | N | **Y** |

## 3. 模块二：内部员工管理

**功能目标**：统一管理授权登录后台的操作账号，不再强制归属部门。

**详细需求：**
1. **员工账号创建**：仅管理员可在员工管理页面创建账号。
   - 必填字段：姓名、登录邮箱、初始密码、角色（管理员 / 授权实施/研发人员）。
2. **列表管理**：支持按姓名、邮箱搜索；管理员可编辑、停用/启用员工账号。
3. **客户关联**：员工与客户解耦，不再展示“名下客户”。任何授权实施/研发人员均可为任意客户生成Key。

---

## 4. 模块三：客户库管理

**功能目标**：建立一个可复用的客户组织名称库，生成Key时直接选用。

**详细需求：**
1. **客户库入口**：管理员和授权实施/研发人员可见“客户库”菜单。
2. **新增客户**：
   - 功能：手动添加客户名称（公司名），唯一。
   - 操作：在客户库页面点击【新增客户】，填写名称保存。
3. **智能联想**：
   - 在生成Key界面，输入客户名称时，系统实时匹配库中已有客户，输入2个字后弹出联想列表。
   - 若库中无匹配项，可直接回车将新名称写入客户库并关联到当前Key（一步完成客户新增与Key生成）。
4. **客户列表**：
   - 展示客户名称、关联Key数量、最近更新时间。
   - 支持按名称搜索。
5. **客户详情**：
   - 点击客户名称可进入详情，列出该客户下所有Key的摘要（Key值部分脱敏显示，状态、额度、创建者、创建时间），并支持跳转至Key管理页面进一步筛选。

## 5. 模块四：Key生命周期管理

### 5.1 生成Key

**生成方式**：支持“单个生成”和“批量生成”。

**入口**：Key管理页面 > 【生成Key】/【批量生成Key】按钮，或从客户详情页快捷进入。

---

#### 5.1.1 单个生成

**生成表单：**

| 字段 | 说明 |
|------|------|
| 客户名称 | 必填，输入框带智能联想。若库中无此名称，回车后自动入库。 |
| Key类型 | 必选：永久Key / 测试Key |
| 额度（元） | 仅测试Key需填写，默认值为2.00元（由系统参数“测试Key默认额度”动态获取），可手动修改，数值>0。永久Key不显示该字段。 |
| 备注 | 选填，用于内部标识。 |

---

#### 5.1.2 批量生成（10-100个）

**操作流程：**

1. 在Key管理页面点击【批量生成Key】，弹出配置窗口。
2. 填写以下配置项：

| 字段 | 说明 |
|------|------|
| 客户名称 | 必填，联想输入，同单个生成规则。 |
| Key类型 | 必选：永久Key / 测试Key |
| 生成数量 | 必填，输入框，限制范围为**10-100**个。 |
| 单个额度（元） | 仅测试Key需填写，默认值**2.00元**（系统参数动态获取），所有批量生成的Key统一使用此额度。永久Key不显示该字段。 |
| 备注 | 选填，所有批量Key共用此备注。 |

3. 点击【确认生成】，系统一次性创建指定数量的Key，所有Key状态默认为“已启用”。
4. **无审批流程，无有效期，无单日生成限额。**

**生成结果展示（全屏结果页）：**

- **结果摘要**：顶部提示“已成功生成 N 个Key，总额度 X 元。”（永久Key不显示总额度）
- **Key清单**：以表格展示，列包含：
  - Key（完整明文显示，提供【复制】按钮）
  - 客户名称
  - Key类型
  - 额度（测试Key显示金额，永久Key显示“无限”）
  - 状态（默认“已启用”）
  - 创建时间
- **一键导出**：
  - 提供【一键导出全部Key信息（CSV）】按钮。
  - 导出文件名：`[客户名称]-Key批量-[生成日期].csv`
  - 文件内容包含完整Key、客户名称、类型、额度、状态、创建时间，无掩码。
  - Key可随时在后台二次查看，不再弹出“无法二次查看”警告。

---

### 5.2 Key管理列表
**功能目标**：提供全局维度的Key检索、筛选与批量操作。

**入口**：Key管理菜单（管理员及授权实施人员可见）。

**列表字段**：
- Key（完整明文展示，提供复制按钮）
- 客户名称
- Key类型（永久/测试）
- 额度（测试Key显示具体金额；永久Key显示“无限”）
- 剩余额度（实时计算）
- 状态（已启用 / 已停用）
- 创建者（内部员工姓名）
- 创建时间
- 操作按钮：【复制】【停用/启用】【删除】（删除仅管理员可见）

**检索与筛选**：
- 支持按Key精确搜索（输入完整Key直接定位所属客户及详情）。
- 支持按客户名称模糊搜索。
- 支持按Key类型、状态、创建者筛选。
- 筛选结果支持**批量勾选**，可批量执行“停用”或“启用”（授权实施人员仅可批量操作自己创建的Key，管理员可操作所有）。

### 5.3 Key停用/启用
- 单个操作：在列表行点击【停用】或【启用】即时生效。
- 批量操作：勾选多个Key后点击【批量停用】或【批量启用】。
- 停用后，该Key发起的API请求将被拒绝，但不删除计费记录。

### 5.4 Key详情与二次查看
- 点击列表中的Key可进入详情页。
- 详情展示：完整Key、客户名称、类型、总额度、已消耗、剩余额度、创建者、状态、创建时间。
- **完整Key始终可见**，不再掩码隐藏，页面提供【复制Key】按钮。
- 可随时查看，不再弹出“无法二次查看”警告。

## 6. 模块五：计费引擎

### 6.1 核心计费逻辑
**原则**：以DeepSeek API返回的usage对象为唯一结算依据，**不加价**。

**计费公式**：
> 扣费金额 = (prompt_tokens × 输入Token单价) + (completion_tokens × 输出Token单价)

**DeepSeek V4 Flash 成本单价（元/百万Tokens）：**
- 输入（缓存命中）：0.02
- 输入（缓存未命中）：1.00
- 输出：2.00

**扣费流程**：
1. 系统转发请求至DeepSeek。
2. 解析Response中的usage.prompt_tokens与usage.completion_tokens。
3. 查询系统配置的模型成本单价。
4. 计算金额并从该次调用使用的Key剩余额度中实时扣除。若为永久Key，则仅记录消耗金额，不设额度限制。

**额度耗尽处理**：
- 测试Key剩余额度不足时，API返回错误提示“额度不足”，并拒绝转发。
- 永久Key无额度校验，正常转发并记录消耗。

### 6.2 模型成本配置
**入口**：系统设置 > 模型管理（仅管理员可见）

**可配置项**：
- 模型名称：DeepSeek V4 Flash
- 模型标识：deepseek-v4-flash
- 输入Token单价（缓存命中 / 未命中）
- 输出Token单价

**售价倍率项已废除**，界面不显示。

## 7. 模块六：安全策略与访问控制

**简化说明**：不再需要复杂的IP黑白名单、域名控制等。仅通过Key状态控制访问。

**实现方式**：
- Key状态为“已启用”时，允许携带该Key的请求正常转发。
- Key状态为“已停用”时，直接拒绝请求，返回“Key已被停用”错误。
- 管理员和授权实施人员可通过管理后台即时停用/启用Key，达到访问控制目的。

## 8. 模块七：数据看板与统计

### 8.1 全局统计看板
**可见权限**：管理员可见全平台数据，授权实施人员仅可见自己生成的Key的汇总数据。

**核心指标卡片**：
- 今日/本周/本月总API调用次数
- 今日/本周/本月总Token消耗
- 今日/本周/本月总金额消耗
- 活跃Key数量（当日有过调用的Key）

**消耗趋势图**：
- 以折线图展示选定时间段内的每日“金额消耗”与“Token消耗”变化。

### 8.2 客户与Key维度报表
**客户用量报表**：
- 以客户为维度，统计各客户的Key数量、活跃Key数、总调用次数、总Token消耗、总金额消耗。
- 支持按金额降序排列。
- 授权实施人员可见全量客户报表（无部门隔离）。

**Key用量明细**：
- 列表展示每个Key的累计调用次数、总Token、总金额，支持按条件筛选。
- 授权实施人员默认仅展示自己创建的Key，管理员展示全部。

## 9. 模块八：运维配置与系统设置

### 9.1 系统参数
**入口**：系统设置 > 参数配置（仅管理员可见）

**可配置项：**

1. **测试Key默认额度**：数值型，单位“元”。  
   - **默认值：2.00**  
   - 影响范围：单个生成和批量生成测试Key时，额度字段的初始默认值。
2. **Key备注模板**（可选）：无实际业务影响。

> **已移除的参数**：试用Key默认有效期、授权员工单日生成额度上限、售价倍率、Key前缀。

### 9.2 操作日志（可选保留）
- 记录Key的生成、停用/启用、删除（管理员）等关键操作，包含操作人、时间、具体Key及客户名称，用于审计。

## 10. 附录：变更对照说明（V1.0 → V2.1）

| 变更点 | V1.0 | V2.1 |
|--------|------|------|
| 角色 | 超级管理员、部门管理员、授权实施/研发、普通员工、外部客户 | 管理员、授权实施/研发人员、客户（后台不可登录） |

| Key检索 | 无 | 支持按Key反查客户、按客户名筛选，批量停用/启用 |
| 计费 | 成本×售价倍率 | 成本价直接扣费，无倍率 |
| 安全策略 | IP黑白名单 | 仅通过Key停用/启用控制 |
| 报表 | 按部门、员工汇聚 | 按客户、Key汇聚；授权人员可见全部数据，不再隔离 |
